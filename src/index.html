Your API Key

            

              Your API key is your unique authentication mean to be used when
              accessing Wordplex’s Phone Number Validation and Verification API.
              To authenticate your requests, you will need to append your API
              key to the base URL as shown below.
            

          

          

            
Base URL

            

              
https://phonevalidation.abstractapi.com/v1/

            

          

          

            

              Overview of the Phone Number Validation and Verification API
            

            

              Abstract's Phone Number Validation and Verification API simply
              requires your unique API key and the phone number you'd like to
              check:
            

            

              https://phonevalidation.abstractapi.com/v1/
  ?
              
                api_key = YOUR_UNIQUE_API_KEY
 
              
              & number = 14154582468
            

            

              This was a successful request, so the valid phone number and
              details associated with it are returned below:
            

            

              {

              "number": "14154582468",

              "is_valid_number": true,

              "local_format": "4154582468",

              "international_format":
              "+14154582468",

              "line_type": "mobile”,

              "registered_location": "Belorus",

              "network": "M1",

              }
            

          

        


        

          
Request Parameters

          

            

              api_key REQUIRED
            

            
String

            

              Note that every Wordplex API has a unique API key, so you will
              need different keys to access the Phone Number Validation and
              Verification, or the Email Validation APIs, for example.
            

          

          

          

            

              phone REQUIRED
            

            
String

            
The phone number to validate and verify.

          

          

        

        

          
Response (json)

          

            
number

            
String

            

              The phone number submitted for verification and validation.
            

          

          

          

            
is_valid_number

            
Boolean

            
Result of the phone format verification.

          

          

          

            
local_format

            
Integer

            

              The number verified in its local, human readable, version.
            

          

          

          

            
international_format

            
String

            

              The number verified in its international, digital dialing ready,
              version.
            

          

          

          

            
line_type

            
String

            

              The type of line associated with the number, that will allow to
              know the type of service that can be used. Note: does not check
              redirections.
            

          

          

          

            
registered_location

            
Boolean

            

              Country of origin and registration of the verified phone number
            

          

          

          

            
network

            
String

            

              Name of the carrier or provider for the verified phone number
            

          

          

        


        

          
Response and Error Codes

          

            Whenever you make a request that fails for some reason, an error
            message is returned also in the JSON format. The message includes an
            error code and a description, which you can find in detail below.
          

          

            
200

            
OK

            

              Everything worked as expected and the phone number verification
              status is returned.
            

          

          

            
400

            
Bad request

            

              The server could not understand the request due to invalid syntax.
              Maybe you forgot a mandatory parameter?
            

          

          

            
403

            
Forbidden

            

              The client does not have access rights to the content; that is, it
              is unauthorized, so the server is refusing to give the requested
              resource. Can happen if you exceeded a reasonable request quota.
